# アルゴリズム学習課題：ソートアルゴリズムの実装

## 1. 実装したアルゴリズム
本課題では、以下の2つのソートアルゴリズムを実装しました。
- **挿入ソート (Insertion Sort)**
- **マージソート (Merge Sort)**

## 2. 実装のポイントと設計の意図

### 挿入ソート
当初は要素を抜き出して挿入する手順で考えていましたが、Python特有の変数入れ替え（スワップ）を用いることで、隣り合う要素を比較しながら適切な位置へ移動させる、より直感的でシンプルな実装に整理しました。

【好きな理由】
「整列済みの山に、新しいカードを適切な位置へ差し込んでいく」という、人間が日常的に行う整理の感覚に最も近く、アルゴリズムの原点を感じるからです。

特に、データが「ほぼ整列している」状態では非常に高速に動作するという特性を知り、状況に応じて最適な手段を選択する重要性を学びました。シンプルでありながら無駄がなく、コードの可読性（読みやすさ）と直感的な理解が両立している点に魅力を感じます。

### マージソート（安定ソートへのこだわり）
【好きな理由】「困難は分割せよ」というデカルトの言葉を体現したような、分割統治（Divide and Conquer）の考え方が非常に美しいと感じるからです。一見複雑に見える大きなデータも、最小単位まで分解すれば単純な比較作業に帰着できるというプロセスは、複雑なプロジェクトを整理して解決に導くマネジメントの考え方にも通じるところがあり、強く惹かれました。また、計算量が安定しており（$O(n \log n)$）、かつ「安定ソート」としてデータの整合性を守るという、実務上の信頼性の高さも好きな理由の一つです。

- **分割統治法の理解**: 複雑な問題を最小単位まで分解してから解決するプロセスを、再帰処理を用いて実装しました。
- **効率的なデータ構造の選択**: リストの先頭要素を取り出す操作（`pop(0)`）が、データ量に応じてパフォーマンスを低下させる点を考慮し、`collections.deque` の `popleft()` を採用しました。これにより、計算量を意識した実装としています。
- **安定性の維持**: 同値のデータの順序が変わらない「安定ソート」の特性を活かすため、マージ時の比較条件を慎重に設定しました。
